1. Overview¶

In the new Data Science profession, researchers are collecting large sets of data, often from Internet sources, and analyzing that data using an amazing array of Python tools. In my Math Majik [Blaa] project, which I presented in the 2021 edition of the NFFS Symposium [Bla21], I showed a bit of Python code that can be used to estimate the model weight and center of gravity location for an indoor model design created using OpenSCAD. In this article, we will extend that work by adding tools you can use to analyze the design and help estimate flight times as well. I think you will be surprised at how easy data analysis can be if you use the modern tools that are hugely popular in today’s research world.

Rather than continue analyzing the Limited Penny Plane design I showed previously, I am going to base this current study on Gary Hodson’s Model of the Year Wart-A6 [Hod10]. Gary is a fellow club member in the Heart of America Free Flight Association, and he has graciously provided flight data and many conversations on his model and his record-setting flights with his design.

I began this study after reading A Method for Predicting Indoor Model Duration by Doug McLean in the 1976 Symposium [McL76].
I also ran into a related study by Walter Erbach who produced a simple program that calculated the power required for an indoor model to maintain level flight [Erb90].

Although predicting flight time, is a worthy goal, my main purpose in presenting this study now is to demonstrate the power of modern tools very popular in the world of Data Science and show you how easy it is to get interesting results on your personal computer.

We will work through several aspects of aerodynamic analysis aimed at predicting flight times, introducing tools and techniques as we proceed. For simplicity, we will focus on 2D aerodynamics. We will leave a 3D study for another time.

If you have some experience with programming, the example code included here should be easy enough to follow. An online version of this study, complete with more details on the code developed, is available at https:rblack42.github.com/nffs-2022-symposium-live/. Those with no programming experience should read the 10 Minute Guide to Python in the Appendix.

Doug drew on extensive research into indoor flight conducted by Max Hacklinger [Hac64], and several classical books on flight that I happen to have in my library. In addition, I will use research by Walter Erbach [Erb90] and Bud Tenny [Ten68] and a few others. We will work through Doug’s development and demonstrate his scheme using some interesting Python tools to conduct the calculations and check the math!

Everything you see in this article can be recreated on you own computer if you are willing to do some setup, or you can play with it using a free web interface.

1.1. Jupyter Notebooks¶

The original document you are reading was developed using a tool very popular in the Data Science world: Jupyter. This is a web based tool that you can install on your personal computer. This program lets you create notebooks where you document your research work, complete with nice figures, math, and code that you can actually run as you work through a development. You cannot easily create a conventional program with Jupyter, but you can write parts of a program that can be copied to a more normal development environment for proper testing and deployment. The program developed as part of this article is available from my GitHub repository [Blab].

If you want to try working through the code I developed for this project, there is a basic guide to getting set up in the appendix (see Setup Guide).

A really neat benefit of working with Jupyter is the existence of free services that let anyone with a web browser experience the research environment I used on my personal laptop. Basically, these free services create a “virtual” computer and then install everything you need on that virtual computer to run a Jupyter web server you can interact with using just your normal web browser. The latest copy of my code gets loaded into this machine from its home on GitHub, and you can experiment with it as you like. You are working on a copy of my code, so you cannot hurt anything. When you close your browser, the virtual machine is destroyed, freeing up the service to support the next person who happens by. I am using the Binder service for this. the project is available at https://mybinder.org/v2/gh/rblack42/nffs-2020-symposium-live/HEAD. A brief overview of using this system is in the appendix (see Binder Guide).

Not all code developed for this project will be presented in this discussion. All code is avaliable at the project Github directory at https://github.com/rblack42/nffs-2022-symposium-live.
